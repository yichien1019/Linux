# 📖 Cloud NAT 維持雲端的獨立性與安全性
* 將 Web Server 放入雲端後，常常會面臨下列兩種問題:
    1. 不想將 Web Server 實際的公開IP位置曝光，以免遭到惡意攻擊
    2. 對外（如三方登入、支付）透過 API 進行連線時，需固定公開網路 IP
## 🔖 VPN Gateway
* 透過 VPN gateway 以保護、授權並處理所有的連線，然而此種方法 VM 仍然使用公開 IP，因此仍有被攻擊的危險性
